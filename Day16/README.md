## Project Title

Customer Database Analysis using SQL

## Project Description
Designed and created a relational database customer_db to store customer information.
Performed SQL queries to analyze customer behavior, spending patterns, and demographics.


## Database Structure
Created a table customers with the following fields:
customer_id
customer_name
city
age
total_spent
number_of_orders

## Key Operations Performed
Inserted sample customer data into the database.
Retrieved records using filtering conditions (WHERE, BETWEEN).
Sorted data using ORDER BY.
Calculated:
Total revenue using SUM()
Average customer spending using AVG()
Grouped data using GROUP BY to analyze:
City-wise spending
Customer count per city
Used HAVING clause to filter grouped results.


## Insights Generated
Identified high-value customers based on spending.
Compared total spending across different cities.
Segmented customers into:
Low (< 10,000)
Medium (10,000 – 50,000)
High (> 50,000)
Determined cities with highest revenue contribution.

## Skills Demonstrated
SQL Database Creation & Management
Data Filtering & Sorting
Aggregation Functions (SUM, AVG, COUNT)
Grouping & Conditional Analysis (GROUP BY, HAVING, CASE)

## Tools Used
MySQL / SQL

## Conclusion
This project demonstrates how SQL can be used to extract meaningful business insights from raw customer data, helping in decision-making and customer segmentation.
